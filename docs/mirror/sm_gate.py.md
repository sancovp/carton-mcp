## Purpose (one paragraph)
`sm_gate.py` is a **scientifically-exact port of CyberneticiRcus's (CCC's) gated state machine** (`cyberneticircus/lib/gates.py` + `lib/state_machines.py` + `neo4j_cypher_mcp/server.py`) onto carton's own `:Wiki` graph. Isaac's mandate (`sm_gate.py:6-9`, verbatim): "IT IS A STATE MACHINE YOU CAN PROGRAM IN CARTON THROUGH ADD CONCEPT AND SET PROPERTIES ... get_concept, EVERY SINGLE OTHER APPLICABLE THING THAT CALLS CYPHER OR RETRIEVES ANY INFORMATION FROM ANYWHERE ARE GATED BY THE STATE MACHINE AND ACTIVATE/CHECK IF THEY ARE LEGAL." The state machine itself is ordinary graph DATA — programmed through carton's own `add_concept`/`set_properties` surface: `State_Machine -[:HAS_STEP]-> Traversal_Step -[:NEXT_STEP{weight}]-> Traversal_Step` (weighted transitions) `-[:CALLS_SM]-> State_Machine` (sub-SM calls); each `Traversal_Step` carries a `required_pattern` regex + instruction `text`. A per-ACTOR cursor (`(actor)-[:HAS_LIFECYCLE]->(Execution_State{status})-[:CURRENT_STEP]->(Traversal_Step)`) is carton's analogue of CCC's Cybernet. **THE GATE:** while an actor is LOCKED at a step, every call's canonical text MUST match that step's `required_pattern` — a match passes AND auto-advances the cursor (to the highest-weight/selected `NEXT_STEP`, a `CALLS_SM` sub-machine, or UNLOCK at a terminal step); a non-match is REFUSED with a `PermissionError` whose message carries the exact regex + instruction (the refusal IS the next-move instruction). Unlike CCC (reads ungated), carton gates BOTH reads and writes. **SAFETY (load-bearing):** the gate is DEFAULT-UNGATED (`gate_call` ALLOWs whenever the actor has no locked `Execution_State`), has a file-based KILL SWITCH (`$CARTON_SM_GATE_DISABLED`), and every gate path is wrapped so a fault FAILS OPEN (logs + ALLOW) — a buggy SM must never lock an agent out of carton. This module is PURE with respect to the live MCP: every graph-touching function takes an injected `run` callable (`(query, params) -> rows`), so it is unit-testable and never imports the server directly; the live wiring lives in `server_fastmcp.py`. Beyond the core gate, the file also builds: an **Agent_Identity ENTITY resolution layer** (identities-are-actual-things, so the gate's per-actor `Execution_State` attaches to a real graph node); a **Sm_Chain** per-concept stack mechanism (a concept's deliberate visit can REQUIRE a specific next move without ever withholding the concept's content); a **skill→SM converter** + a **generic SM-creation factory** (`create_sm_chain`) that programs an arbitrary ordered stack of SMs from a spec, now with a real **BRANCHING** decision mechanism (`select_branch`, softmax-weighted, with **weight reinforcement** learning from which routes actually get taken); and a **graph→system-prompt generator** (`generate_system_prompt`) that assembles the live GNOSYS system prompt FROM the graph rather than from static text. **THIS FILE'S TWO P0 EXHAUST-LEDGER ADDITIONS (Griess-Neural-Surrogate patch, 2026-07-06):** an `episode_ledger_path`/`_record_sm_episode` append-only JSONL capturing every SM-gate trajectory event (CCC's `bandit_choices`, previously entirely lost), and an `actor` parameter threaded through `gate_call` into `auto_progress` so every recorded episode carries who acted.

## Notes
1. **The module is PURE w.r.t. the live MCP.** Every function that touches the graph takes an injected `run: Callable[[str, Dict], List[Dict]]` — it never imports `server_fastmcp` or opens its own connection except in the explicitly-named `*_live` self-contained entries (`resolve_identity_entity_live`, `create_sm_chain_live`), which exist specifically so external callers in OTHER processes (dragonbones, `equip_persona`) never manage a neo4j connection themselves.
2. **Every gate/trigger/visit path FAILS OPEN.** A buggy state machine, a malformed regex, a missing node, a connection fault — all degrade to `{"allowed": True}` / `require_next: None` / no lock, logged loudly. The ONLY place this file raises deliberately is `GateRefusal` — an intentional, correctly-functioning refusal (illegal move / no eligible branch), never a fault.
3. **The P0 episode ledger (`_record_sm_episode`) is best-effort exhaust, never a gate.** It wraps its own file I/O in try/except and swallows failures — recording a trajectory event must never affect whether a call is allowed or refused.
