## Purpose (one paragraph)
`exhaust_records.py` declares the **canonical Pydantic shapes** of the three P0 exhaust-ledger record types carton writes as a byproduct of normal operation (the "Griess-Neural-Surrogate exhaust patches," 2026-07-06): `soma_rejections.jsonl` (written by `add_concept_tool.record_soma_rejection` — oracle-labeled hard negatives, i.e. claim-structures SOMA's verdict rejected or flagged), `sm_episodes.jsonl` (written by `sm_gate._record_sm_episode` — SM-gate trajectory events, the direct analogue of CyberneticiRcus's `bandit_choices`), and `soma_fired_chains.jsonl` (written by `add_concept_tool`'s `fired_chains=` verdict-block parse — which deduction chains fired for a given event, the training substrate for a future `Chain_Prioritizer`). This module holds the model DEFINITIONS ONLY — it is not itself a ledger writer. Per the module docstring (`:8-13`), these models are what `gnosys-vault` (`gnosys_vault/carton_exhaust.py`) `vault()`s into SOMA so the three exhaust-record shapes become DEFINED SYSTEM TYPES, per the vault keystone doctrine that code is the only type source. The three ledger-WRITING functions (`record_soma_rejection` in `add_concept_tool.py`, `_record_sm_episode` in `sm_gate.py`) deliberately stay plain best-effort `dict` appends — a pydantic import in the hot gate/validation path buys nothing there — and the coherence between each writer's dict shape and the model declared here is pinned by `tests/test_p0_exhaust_ledgers.py`'s shape-assertion tests; the docstring is explicit that if a writer gains or renames a key, the model here must be updated in the same change.

## Surface (1:1 — every public thing, in file order)

- `class SomaRejectionRecord(BaseModel)` — `exhaust_records.py:20-31`
  - Fields: `concept: str`, `relationships: List[Dict[str, Any]]`, `verdict_kind: str`, `timestamp: str`, `reason: str = ""`.
  - One oracle-labeled hard negative: a claim-structure whose SOMA verdict rejected or flagged it. `verdict_kind` is `'contradiction'` (a Type-2 geometric reject — carton never saved the node) or `'mereo_error'` (a Type-1 undefined-is_a fill signal — carton DOES save the node as soup, but the record is still a negative example of a well-formed claim). Its docstring names this as the training gold a `Slot_Fill_Ranker` needs — the kind of hard negative most KG-completion projects have to FAKE by corrupting real triples; here it is captured for free as a byproduct of normal `add_concept` operation.
  - Matches the dict `record_soma_rejection` writes in `add_concept_tool.py` (`concept`, `relationships`, `verdict_kind`, `reason`, `timestamp` — same five keys, same names).

- `class SmEpisodeRecord(BaseModel)` — `exhaust_records.py:34-54`
  - Fields: `event: str`, `timestamp: str`, and nine OPTIONAL fields (all default `None`): `actor: Optional[str]`, `state_id: Optional[str]`, `curr_step: Optional[str]`, `chosen: Optional[str]`, `candidates: Optional[List[Dict[str, Any]]]`, `call_text: Optional[str]`, `required_pattern: Optional[str]`, `reinforce_delta: Optional[float]`, `concept: Optional[str]`, `sm_id: Optional[str]`, `entry_step: Optional[str]`.
  - One SM-gate trajectory event (the CCC `bandit_choices` analogue). `event` names the trajectory-event kind: `lock` / `lock_trigger` / `branch_chosen` / `advance_explicit` / `terminal_unlock` / `refusal_no_branch` / `refusal_pattern`. The docstring calls out `branch_chosen` specifically — it carries the full decision context (every candidate with its pattern+weight, the call text, the pick, the reinforcement applied) — as the `Policy_Network_Sm_Selector` training substrate. Because every field beyond `event`/`timestamp` is optional, this ONE model covers all seven event kinds without a union/subclass per kind — a given record only populates the fields relevant to its own `event` value (e.g. `refusal_no_branch` populates `candidates`/`call_text` but not `chosen`/`reinforce_delta`).
  - Matches the dict shapes `_record_sm_episode` writes in `sm_gate.py` across its seven call sites (`lock`, `lock_trigger`, `branch_chosen`, `advance_explicit`, `terminal_unlock`, `refusal_no_branch`, `refusal_pattern`) — every key any of those seven call sites passes is present here as an optional field.

- `class FiredChainsRecord(BaseModel)` — `exhaust_records.py:57-68`
  - Fields: `concept: str`, `fired_chains: List[str]`, `timestamp: str`, `status: Optional[str] = None`.
  - One event's fired-deduction-chain NAMES (from SOMA's `fired_chains=` verdict block) — WHICH chains fired for the event that validated `concept`, not just a count, per the docstring the `Chain_Prioritizer` training substrate. `status` carries the concept's own verdict level when known (`soup`/`code`/`mereo_error`/...).
  - Matches the dict `add_concept_tool_func`'s `fired_chains=` parse appends to `soma_fired_chains.jsonl` (`concept`, `fired_chains`, `status`, `timestamp` — same four keys, same names).

## Notes
- **This module has NO writer logic of its own.** It is pure declaration — three Pydantic classes, nothing else runs at import or call time. The actual JSONL-append writers (`record_soma_rejection`/`rejection_ledger_path` in `add_concept_tool.py`; `_record_sm_episode`/`episode_ledger_path` in `sm_gate.py`; the inline `fired_chains=` parse-and-append block in `add_concept_tool_func`) live in their respective modules and were a deliberate choice to stay plain-dict, per the module docstring: "a pydantic import in the hot gate path buys nothing."
- **The coherence between a writer's dict and its model here is enforced ONLY by `tests/test_p0_exhaust_ledgers.py`'s shape assertions — not by any runtime validation.** If a writer adds or renames a key without a matching change here (and without that test being re-run), the drift will not be caught until `gnosys_vault.carton_exhaust`'s `vault()` call fails on the mismatched shape, or silently drops the new field (Pydantic's default behavior is to ignore extra kwargs unless configured otherwise — UNVERIFIED whether `carton_exhaust.py` configures strict/forbid-extra mode; check that file directly before assuming a silent drop vs a hard failure).
- **`SmEpisodeRecord`'s "everything but `event`/`timestamp` is optional" design is a discriminated-union-by-convention, not an enforced one** — nothing in this file asserts that a `refusal_pattern` record must carry `required_pattern`/`call_text` and must NOT carry `chosen`/`reinforce_delta`; that shape discipline lives entirely in `sm_gate.py`'s seven `_record_sm_episode` call sites. A reader parsing `sm_episodes.jsonl` back into this model gets a validly-typed but permissively-shaped record regardless of which `event` kind it is.
